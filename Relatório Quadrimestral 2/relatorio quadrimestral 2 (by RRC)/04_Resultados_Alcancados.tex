\subsection{Eletrônica}

A partir da primeira solução conceitual para a eletrônica, foram projetadas duas PCBs. Para o projeto, utilizou-se o software Altium, que permite gerar esquemáticos e modelos 3D com roteamento. Uma das placas tem como objetivo garantir robustez através de redundância de micro controladores, enquanto que a outra foca em simplicidade, entrega rápida e de menor custo. Alguns componentes da placa já estão disponíveis em laboratório e foram testados, outros estão na lista de componentes que serão adquiridos ainda até fim de maio.

A segunda solução para a eletrônica embarcada, que envolve a utilização de um PC, resultou na pesquisa de fornecedores de PC104 com placas ADC e CAN. Drivers de diversos dispositivos já estão disponíveis em ROCK, o que facilita a execução desta solução tanto para software quanto para hardware.
Por último, em Maio, foi montado um protótipo da eletrônica embarcada para os testes em Jirau. Serão utilizadas as interfaces RS232-USB e analógica para os testes do sonar Micron (disponível em laboratório) e sensores indutivos. A alimentação será realizado por um pack de baterias 12V, 7Ah em série. A proposta de soluções para a arquitetura da eletrônica do projeto ROSA encontra-se no Relatório de Eletrônica.

\subsection{Reconstrução 3D por sonar}

O princípio por trás da parte de reconstrução 3D do projeto é  acumular os dados provenientes do sonar em uma representação volumétrica. Esse acúmulo ajuda a resolver o problema de ambiguidade dos dados de sonar. O framework Octomap foi definido durante a fase de definição do projeto como a biblioteca a ser utilizada para o acúmulo volumétrico. Logo, o Octomap foi integrado ao framework Robot Construction Kit (ROCK), framework de robótica utilizado no projeto. A integração foi testada em laboratório utilizando-se um sonar Tritech Micron. A figura \ref{fig:octomap} ilustra o preenchimento volumétrico de acordo com a forma de onda proveniente do sonar e a intensidade de reflexão.

\subsection{Interface}

A interface com o usuário facilita a utilização do robô ROSA, permitindo a compreensão das informações geradas pelo robô de modo intuitivo.
Neste quadrimestre, o protótipo da interface foi desenvolvido e implementado em um tablet Android para análise de usabilidade, como ilustrado na figura \ref{fig:userinterface}.

A prototipagem é uma técnica de análise interativa em que os usuários estão ativamente envolvidos no desenvolvimento da interface.
A descrição detalhada do protótipo de interface encontra-se no Relatório de Interface de Usuário.

\subsection{Drivers}

Os drivers permitem a comunicação entre o computador e os devices conectados ao mesmo.
Neste quadrimestre, foram encomendados os sensores especificados no projeto básico. Dentre eles, somente o sensor indutivo, encoder e profundímetro foram entregues.
Os drivers para os dispositivos entregues foram desenvolvidos e integrando ao Framework de robótica ROCK.
